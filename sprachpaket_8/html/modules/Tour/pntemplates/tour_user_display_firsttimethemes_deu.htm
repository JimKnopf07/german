Themes

Alles eine Frage des Aussehens

Mit dem PostNuke Theme-System können Themeentwickler eine breite Vielzahl von Themes - vom traditionellen zum sehr einzigartigen - errichten. Das Theme-Modul, auch bekannt als Xanthia, ist benutzerfreundlicher als die vorhergehenden Versionen. Themes bieten dem Administrator die Möglichkeit, das Aussehen des Internetauftritts mit nur wenigen Mausklicks umzugestalten. Die Distributionen werden mit einer gewissen Vorauswahl von Themes ausgeliefert. Weitere Themes stehen, z.B. in der Themedatenbank postnuke.com Zur Verfügung.

Trennung von Inhalt und Layout

PostNuke erlaubt nicht nur um Platzierung von Inhalten in Themezonen, sondern ermöglicht weiterhin die vollständige Neugestaltung der Seite durch modulspezifische Templates. Themezonen werden eingesetzt, um verschiedenen Blöcken oder Modulen ein bestimmtes aussehen zu geben. Jeder Themezone ist ein Template zugeordnet, die Themezone wiederum einem Block oder einem Modul. Somit ist man der Lage, eine gut strukturierte Darstellung aller Daten zu definieren. Weiterhin kann sogar ein Datenmix aus verschiedenen Modultypen erstellen werden.
Wenn die vorhandenen Modultemplates nicht gefallen, können diese ohne großen Aufwand modifiziert werden und im Themeverzeichnis abgelegt werden. Somit werden diese auch nach einem Upgrade nicht überschrieben.

Smarty

Diese Funktionen werden in PostNuke über die Smarty Template Engine realisiert. Somit stehen viele mächtige PHP-Funktionen über einfaches HTML zur Verfügung. Es müssen nur wenige neue Tags gelernt werden, die die vorhandenen Möglichkeiten bei der Erstellung von eigenen Themes sinnvoll bereichern.
