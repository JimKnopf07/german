Das “Adam-Baum”-Release

PostNuke ist ein Open Source Application Framework, das Inhalte von Design und Technik trennt. Die Inhalte einer Internetpräsenz (zum Beispiel Beiträge, Links, Downloads, FAQs, Bildergalerien, Foren etc.) können dabei direkt via Browser verwaltet werden. Durch die klare Aufgliederung in Form, Funktion, Inhalt und Gestaltung hilft PostNuke, die Kosten und den Aufwand beim Betrieb einer Webseite zu reduzieren.

PostNuke ist modular aufgebaut - die zentralen Funktionen (Benutzermanagement, Berechtigungssystem, API) werden vom sogenannten Core übernommen, der Leistungsumfang kann darüberhinaus durch Zusatzmodule individuell angepasst und nahezu beliebig erweitert werden, unter anderem auch zu einem sehr leistungsfähigen Content Management System (CMS).

PostNuke hat eine sehr große Community und bietet ausgezeichneten Support in den Foren. Die deutsche Anlaufstelle rund um Dokumentation und Support findet man unter http://support.pn-cms.de. Wir begrüßen Beiträge von jedem, unabhängig von seinen Kenntnissen in PostNuke. Wie man helfen kann.

Neue Installation

Es gibt zwei Möglichkeiten PostNuke zu installieren:

  1. Webbrowser-basierte Installation (empfohlen)
  2. manueller Import des SQL-Dumps

1. Die Webbrowser-basierte Installation (empfohlen)

Der einfachste Weg zu einer neuen Installation ist sicherlich die Installationsroutine unter der Adresse “install.php”. Die exakte URL hängt von der jeweiligen Domain ab, aber wenn die Seite zum Beispiel “http://www.example.com” ist, lautet die URL “http://www.example.com/install.php”. Wenn PostNuke in ein Unterverzeichnis "pn" kopiert wurde, lautet die URL “http://www.example.com/pn/install.php”. Das Installations-Skript führt Schritt für Schritt durch die gesamte Installation, der Vorgang dauert insgesamt nur etwa 5 bis 10 Minuten.

HINWEIS

PostNuke benötigt während der Installation Zugang zu einer Datenbank. Im sogenannten Shared Hosting stellt oftmals der Provider die Informationen über Datenbank-Benutzer, Datenbank-Passwort und den Datenbank-Namen zur Verfügung (in diesem Fall legt die Installationsroutine nur entsprechende Tabellen an), bei vollem Zugriff auf den Server - und wenn der Datenbank-Benutzer über entsprechende Berechtigungen verfügt - kann auch während der Installation eine Datenbank angelegt werden.

Nachdem die Installation abgeschlossen ist, kann die PostNuke Seite ganz einfach über den Webbrowser aufgerufen werden (z.B. http://www.example.com).

2. Installation durch manuellen Import eines SQL-Dumps

Die zweite Möglichkeit der Installation ist ein direktes Einspielen eines SQL-Dumps in eine bestehende Datenbank - diese Möglichkeit ist aber nicht einfach durchzuführen und sollte nur von erfahrenen Benutzer in Betracht gezogen werden.

HINWEIS

Der SQL-Dump ist aus einer MySQL 5.x Datenbank erstellt worden, und wird nicht mit einer MySQL Version vor 5.x oder einem anderen Datenbanktyp als MySQL funktionieren.
Nachdem der Dump in die Datenbank importiert wurde, muß die Datei /config/config.php angepasst werden, die erforderlichen Werte für den Benutzernamen und das Passwort können dabei optional Base64-enkodiert eingetragen werden ($PNConfig['DBInfo']['default']['encoded'] = 1;).
Das Login nach manuellen Import des Dumps ist mit dem Benutzernamen “Admin” und dem Kennwort “Password” (beides ohne “”) möglich, aus Sicherheitsgründen sollten diese Defaultwerte nach der Installation für den laufenden Betrieb angepasst werden.

Abschließende Anmerkung

Direkte Modifizierungen (sog. “Hacks”) am Systemcode oder der Datenbank werden nicht unterstützt, für individuelle Modifizierungen kann auch kein Support geleistet werden. Anpassungen können darüberhinaus bei zukünftigen Aktualisierungen auch umfassende Probleme verursachen. Als Alternative empfehlen wir, sich die Entwickler Dokumentation anzusehen und die Modularität des Systems zukunftssicher auszunutzen.
Für Fragen rund um die Installation, die Einrichtung und den Betrieb steht unter support.pn-cms.de auch eine umfangreiche deutschsprachige Seite zur Verfügung.

Wir wünschen viel Spaß mit dieser Version von PostNuke!
