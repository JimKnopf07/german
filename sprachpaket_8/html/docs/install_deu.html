PostNuke: Das adam_baum Release

PostNuke ist ein Open Source Content Management System (CMS), das Inhalte von Design und Technik trennt. Die Inhalte einer Internetpräsenz (zum Beispiel Beiträge, Links, Downloads, FAQs, Bildergalerien, Foren etc.) können dabei direkt via Browser verwaltet werden. Durch die klare Aufgliederung in Form, Funktion, Inhalt und Gestaltung hilft PostNuke, die Kosten und den Aufwand beim Betrieb einer Webseite zu reduzieren. PostNuke ist modular aufgebaut - die zentralen Funktionen (Benutzermanagement, Berechtigungssystem, API) werden vom sogenannten Core übernommen, der Leistungsumfang kann darüberhinaus individuell angepasst und nahezu beliebig erweitert werden.

PostNuke hat eine sehr große Community und bietet ausgezeichneten Support in den Foren. Die Deutsche Community mit Dokumentation, Support und Download von PostNuke finden Sie unter http://support.pn-cms.de.

Wir begrüßen Beiträge von jedem, unabhängig von seinen Kenntnissen in PostNuke. Wie Sie helfen können..

Neue Installation

Es gibt zwei Möglichkeiten PostNuke zu installieren:

  1. Die Web-Browser-basierte Installation.
  2. oder über einen manuellen Import des SQL dumps.

1. Die Web-Browser-basierte Installation

Starten Sie die Installation, indem Sie die “install.php” mit Ihrem Browser öffnen. Die exakte Url hängt von der jeweiligen Domain ab, aber wenn die Seite zum Beispiel “http://www.example.com” ist, so lautet die Url http://www.example.com/install.php.

Wen Sie PostNuke in ein Unterverzeichnis "pn" kopiert haben, so lautet diese Url http://www.example.com/pn/install.php

Das Installationsscript führt Sie Schritt für Schritt durch die gesamte Installation. Dauer dieses Vorgangs etwa 5 bis 10 Minuten

ACHTUNG

PostNuke benötigt bevor Sie das Installationsscript aufrufen eine Datenbank. Erstellen Sie bitte eine Datenbank. Dann notieren Sie sich den Datenbanknutzer, das Datenbank Passwort und den Datenbanknamen. Sie werden diese Angaben im Laufe der Installation benötigen.

Nachdem die Installation abgeschlossen ist, können Sie ihre PostNuke Seite ganz einfach mit ihren Webbrowser aufrufen.(z.B. http://www.example.com)

2. Manueller Import des SQL dumps (Nicht empfohlen)

Die zweite Möglichkeit der Installation von PostNuke ist ein manuelles hinzufügen des SQL dumps. Diese Möglichkeit ist aber nicht einfach durchzuführen und sollte nur von erfahrenen PostNuke Benutzer durchgeführt werden. Der MySQL Dump ist mit einer MySQL 5.x Datenbank erstellt worden, und wird nicht mit einer MySQL Version vor 5.x oder einen anderen Datenbank Typ funktionieren. Nachdem Sie den dump ihrer Datenbank hinzugefügt haben, müssen sie noch die config.php bearbeiten. Die Werte für den Benutzernamen und das Passwort sollten aus Sicherheitsgründen base64 decodiert werden. (Freie Tools hierfür findet man im Internet)

Hinweis:

Der Name zum Login ist hierbei "Admin" und das Passwort ist "Password" (beides ohne "")

Abschließende Anmerkung

Modifizierungen "´hacks" am PostNuke Systemcode oder der Datenbank werden nicht unterstützt. Auf diese Modifizierungen wird auch kein Support geleistet. Diese Modifizierungen können bei einen zukünftigen Upgrade umfassende Probleme verursachen, und deshalb werden diese 'hacks' nicht empfohlen. Wir empfehlen stattdessen, dass Sie sich einmal die Entwickler Dokumentation ansehen.
